工作原理
远端浏览器安装 OpenClaw Browser Bridge 插件后，插件会通过 WebSocket 主动连接到本服务器。 相比传统的 --remote-debugging-port 方案，插件方式：
✅ 不需要命令行启动 Chrome
✅ 不需要在远端开放任何端口
✅ 插件主动连接服务端，可穿越 NAT/防火墙
✅ 使用配对码认证，更安全
安装步骤
步骤 1：获取插件
在本项目的 web/browser-extension/ 目录中包含了完整的插件源码。
步骤 2：加载插件到 Chrome
① 在远端机器的 Chrome 地址栏输入 chrome://extensions 并回车
② 打开右上角 「开发者模式」开关
③ 点击 「加载已解压的扩展程序」
④ 选择 web/browser-extension/ 文件夹
⑤ 插件加载后，工具栏会出现绿色圆形图标
步骤 3：配置连接
① 在上方「配对码管理」中点击 「生成新配对码」，获得 6 位配对码
② 点击远端 Chrome 工具栏的插件图标，打开弹窗
③ 填入本服务器地址（如 http://192.168.x.x:3000）
④ 填入配对码
⑤ 点击「连接」，状态变为绿色即成功
步骤 4：验证
连接成功后，上方「已连接的浏览器」区域会显示该设备。 点击「测试」按钮可以获取远端浏览器的标签页列表，确认控制链路正常。
💡 安全说明：插件与服务端之间使用配对码认证，连接仅在局域网内建立。 删除配对码会立即断开对应的浏览器连接。建议仅在可信的局域网环境中使用此功能。

插件市场
管理 Skills（知识包）和 Extensions（功能扩展），增强 OpenClaw 的能力
📖 Skills — Markdown 驱动的知识包，为 Agent 注入专业领域能力（如 GitHub 操作、Notion 读写等）。每个 Skill 是一个包含 SKILL.md 的目录。
🧩 Extensions — TypeScript 插件，扩展 OpenClaw 运行时功能（如新的消息通道、工具集、存储后端等）。通过 npm 安装。
所有已安装 Skills（内置 + 扩展 + 手动安装至 ~/.openclaw/skills/）。含安全扫描标记。
扫描 Skill 源
远程仓库：输入 URL 后点扫描（如 https://github.com/user/skills-repo）。本地目录：点 📂 按钮选择文件夹，含客户端安全扫描（支持 Chrome/Edge/Safari）。
